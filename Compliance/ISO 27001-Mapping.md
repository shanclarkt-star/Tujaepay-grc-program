# ISO/IEC 27001 Control Mapping

## Purpose
This document maps TujaePay security controls to ISO/IEC 27001 Annex A requirements to demonstrate alignment with international information security standards.

## Scope
This mapping applies to TujaePay systems, processes, and data supporting the SaaS platform.

---

## ISO 27001 Control Mapping Table

| ISO Control | Control Description | TujaePay Implementation | Evidence |
|------------|--------------------|-------------------------|----------|
| A.5.1 | Information security policies | Security policies approved by management | Information Security Policy |
| A.6.1 | Security roles and responsibilities | Defined governance roles | Risk Management Charter |
| A.8.2 | Information classification | Data classified by sensitivity | Data classification section |
| A.9.1 | Access control | Least privilege and RBAC enforced | Access Control Policy |
| A.9.4 | Secure log-on procedures | MFA required for production systems | IAM configuration |
| A.12.4 | Logging and monitoring | Centralized logging enabled | Monitoring procedures |
| A.16.1 | Incident management | Formal incident response process | Incident Response Policy |
| A.15.1 | Supplier relationships | Vendor risk assessments conducted | Vendor reviews |

---

## Control Gaps
Any identified control gaps are documented, risk-rated, and tracked in the TujaePay risk register until remediation is complete.

## Review
This document is reviewed annually or prior to ISO 27001 audits.
